What to set up
before each exercise

A clear, step-by-step setup guide for all 24 exercises. Know exactly what to install — and what you can skip — before you start.

✓ No Setup — use as-is ⚙ Quick Install — apt/yum 🖥 VM Required — needs virtual disks 🐋 Use Docker — safest option

💡 Short Answer

Easy exercises (1–8): Mostly run on any Linux machine with zero setup. A couple need one quick apt install.
Medium exercises (9–16): Some need a service installed (MySQL, Nginx, Node.js). Disk-based exercises (LVM, RAID) need a VM with extra virtual disks.
Hard exercises (17–24): Use Docker or a dedicated VM — these involve complex services (containers, firewalls, backup daemons, live kernel patching).

EASY
Exercises 1 – 8: Fundamentals
Most require zero setup. Just open a terminal.
Ex 01 User Management for New Team Members No Setup
All commands (useradd, passwd, groupadd) are built into every Linux system. Just open a terminal and start.
Verify your system has these tools
which useradd groupadd passwd chage # all should return paths
Ex 02 File Permission Fix for Web Server No Setup
Uses chmod, chown, find — all built-in. Create a practice folder to simulate the web directory safely.
Create a safe practice directory
sudo mkdir -p /var/www/myapp/logs sudo groupadd developers 2>/dev/null sudo useradd -r www-data 2>/dev/null # usually already exists
Ex 03 Disk Space Cleanup Investigation No Setup
Uses df, du, find, gzip — all built-in. Run directly on your own system to practice finding real large files.
Ex 04 Process Management for High CPU Usage 1 Install
top, ps, kill are built-in. Install htop for a friendlier experience (the hints mention it).
Install (optional but recommended)
sudo apt install htop lsof -y
Ex 05 Cron Job Setup for Automated Backups Quick Install
Cron is built-in. You need MySQL (for mysqldump) and mailutils for email alerts. You can skip email for basic practice.
Install MySQL + mail tools
sudo apt install mysql-server mailutils -y sudo systemctl start mysql sudo mkdir -p /backup/db
Tip: If you only want to practice cron syntax and don't need actual DB backups, skip MySQL and just create a dummy script that echoes "Backup done" — the scheduling part is identical.
Ex 06 Network Connectivity Troubleshooting Quick Install
Install the network diagnostic tools. You can test against localhost or any public server instead of db.company.com.
Install network tools
sudo apt install telnet net-tools iputils-ping -y # ss and ping are usually pre-installed # Test against localhost: telnet localhost 3306
🐋 Optional: Spin up a real target to test against
docker run -d -p 3306:3306 -e MYSQL_ROOT_PASSWORD=test --name db mysql:8 # Now test: telnet localhost 3306
Ex 07 SSH Key-Based Authentication Setup Quick Install
You need an SSH server to connect to. The easiest approach: install OpenSSH server on your own machine and SSH into localhost.
Install SSH server (if not already installed)
sudo apt install openssh-server -y sudo systemctl enable --now ssh # Now you can SSH to: ssh your-username@localhost
Ex 08 System Resource Monitoring Setup Quick Install
vmstat and iostat need sysstat. Network monitoring needs iftop and nethogs.
Install monitoring tools
sudo apt install sysstat iftop nethogs -y sudo systemctl enable --now sysstat # Verify: sar -u (should show CPU data after a few minutes)
MEDIUM
Exercises 9 – 16: System Administration
Some need apt installs. Disk exercises (LVM, RAID) need a VM with extra virtual disks.
Ex 09 LVM Setup for Dynamic Storage VM + Virtual Disks
You need 3 extra virtual disks (sdb, sdc, sdd). This only works in a VM where you can add disks. Use VirtualBox or VMware and add 3 x 10GB disks to your Linux VM.
After adding virtual disks in your VM settings
sudo apt install lvm2 -y lsblk # confirm /dev/sdb, sdc, sdd appear
VirtualBox setup: Machine → Settings → Storage → Add Hard Disk → Create 3 new VDI disks (10GB each). Reboot the VM. They'll appear as /dev/sdb, /dev/sdc, /dev/sdd.
Ex 10 Systemd Service for Custom Application Quick Install
Needs Node.js to create a simple app to manage as a service. Systemd itself is already on your system.
Install Node.js and create a minimal test app
sudo apt install nodejs -y sudo useradd -r -s /bin/false appuser sudo mkdir -p /opt/myapp echo "const http=require('http'); http.createServer((_,r)=>{r.end('ok')}).listen(3000)" | sudo tee /opt/myapp/server.js sudo chown -R appuser:appuser /opt/myapp
Ex 11 Nginx Web Server Configuration Quick Install
Install Nginx. Everything else (config files, virtual hosts) is part of the exercise itself.
Install Nginx
sudo apt install nginx -y sudo systemctl enable --now nginx curl http://localhost # should return Nginx welcome page
Ex 12 Network Performance Tuning with sysctl No Setup
Everything is kernel parameters edited via /etc/sysctl.conf. No software to install — just make sure you have sudo access.
Confirm you can read/write sysctl settings
sudo sysctl -a | head -5 # should show kernel params cat /proc/sys/vm/swappiness # should show a number
Ex 13 RAID Configuration for Data Redundancy VM + Virtual Disks
Needs 4 extra virtual disks (sdb, sdc, sdd, sde). Same approach as LVM — add them in your VM settings.
After adding 4 virtual disks in VM settings
sudo apt install mdadm -y lsblk # confirm 4 new disks appear
Ex 14 Log Rotation and Centralized Logging Quick Install
logrotate is usually pre-installed. You need rsyslog for centralized logging and a second machine (or Docker container) as the log server.
Install and verify
sudo apt install rsyslog -y sudo systemctl enable --now rsyslog which logrotate # should return /usr/sbin/logrotate
🐋 Simulate a central log server with Docker
docker run -d -p 514:514/udp --name syslog-server \ -e SYSLOG_CONF="*.* /var/log/remote.log" rsyslog/syslog_appliance_alpine
Ex 15 Firewall Configuration with iptables/ufw Quick Install
iptables is built-in. Install ufw for the simpler frontend. Practice on a VM — not on a remote server you can't access physically.
Install ufw
sudo apt install ufw -y sudo ufw status # should show: Status: inactive # Important: enable SSH rule BEFORE enabling ufw sudo ufw allow ssh
⚠ Warning: Always run sudo ufw allow ssh before sudo ufw enable or you will lock yourself out of a remote machine.
Ex 16 MySQL / Database Administration Quick Install
Install MySQL Server. The exercise covers performance tuning, user management, and query optimization.
Install and secure MySQL
sudo apt install mysql-server -y sudo systemctl enable --now mysql sudo mysql_secure_installation # follow prompts mysql -u root -p # verify login works
HARD
Exercises 17 – 24: Advanced Administration
Use Docker or a dedicated VM. These involve complex, system-level changes.
Ex 17 Docker Container Management Install Docker
This exercise IS Docker — install it and you're ready to go.
Install Docker Engine (Ubuntu)
sudo apt install docker.io docker-compose -y sudo systemctl enable --now docker sudo usermod -aG docker $USER # run docker without sudo newgrp docker docker run hello-world # confirm it works
Ex 18 SELinux / AppArmor Security Policies Quick Install
Ubuntu uses AppArmor by default (already installed). Install the management tools to interact with it.
Install AppArmor tools
sudo apt install apparmor-utils apparmor-profiles -y sudo aa-status # shows current AppArmor profiles sudo systemctl enable --now apparmor
Ex 19 Nginx Load Balancer Setup Use Docker
Needs multiple backend servers. Docker makes this trivial — spin up 3 containers as backends.
🐋 Spin up 3 backend servers instantly
docker run -d -p 8081:80 --name backend1 nginx docker run -d -p 8082:80 --name backend2 nginx docker run -d -p 8083:80 --name backend3 nginx # Then configure Nginx on your host to load-balance across :8081 :8082 :8083
Install Nginx (if not already done in Ex 11)
sudo apt install nginx -y
Ex 20 Kubernetes (k8s) Basic Cluster Setup Use minikube
Use minikube — it runs a full single-node Kubernetes cluster locally without needing multiple VMs.
Install kubectl + minikube
sudo apt install kubectl -y curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64 sudo install minikube-linux-amd64 /usr/local/bin/minikube minikube start --driver=docker kubectl get nodes # should show 1 node: Ready
Ex 21 Performance Profiling with perf/strace Quick Install
Install Linux perf tools and strace. These work on your own system — profile any running process.
Install profiling tools
sudo apt install linux-tools-common linux-tools-generic strace ltrace -y sudo perf stat ls # quick test — should output performance counters
Ex 22 Ansible Automation for Configuration Management Use Docker
Ansible needs target machines to configure. Use Docker containers as your "fleet" of servers.
Install Ansible
sudo apt install ansible -y ansible --version # confirm installation
🐋 Create a fleet of target servers with Docker
docker run -d --name server1 -p 2221:22 rastasheep/ubuntu-sshd:18.04 docker run -d --name server2 -p 2222:22 rastasheep/ubuntu-sshd:18.04 docker run -d --name server3 -p 2223:22 rastasheep/ubuntu-sshd:18.04 # SSH password is: root # Use localhost:2221, :2222, :2223 as your inventory
Ex 23 Bacula Enterprise Backup System Use Docker
Bacula has multiple components (Director, Storage, File daemon). Docker Compose makes this manageable without complex manual setup.
🐋 Recommended: Docker Compose setup
sudo apt install bacula bacula-console -y # OR use a pre-built Docker image: docker run -d --name bacula \ -p 9101:9101 -p 9102:9102 -p 9103:9103 \ baculasystems/baculaweb
Note: This is one of the most complex exercises. If you're new to backup systems, spend 30 minutes reading the Bacula documentation first before attempting setup.
Ex 24 Zero-Downtime Kernel Patching with kpatch Dedicated VM Only
This is the most complex exercise. kpatch requires kernel source code matching your exact running kernel. Do this in a throwaway VM only — never on a machine you care about.
Setup (inside a dedicated VM)
sudo apt install kpatch kpatch-build -y # Or build from source (required for custom patches): sudo apt install git make gcc libelf-dev -y git clone https://github.com/dynup/kpatch.git cd kpatch && make && sudo make install kpatch --version
⚠ Important: kpatch needs kernel headers matching your exact kernel version. Run uname -r and install linux-headers-$(uname -r). If versions don't match, kpatch-build will fail. Use a fresh Ubuntu VM where you control the kernel version.
